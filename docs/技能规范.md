# 技能规范

## 目标

技能是用户进入某类任务的产品入口，不是单纯的提示词，也不是 MCP 工具本身。技能规范需要支持内置技能、用户自定义技能、组织内部分发技能和市场技能持续发布。

本规范的目标：

- 让技能可以被发现、安装、配置、运行、升级和下架。
- 让每个技能有明确的任务边界、启动工作区、模型配置、工具能力和权限说明。
- 兼容现有 `Skill` 数据结构，避免一次性重构破坏当前会话。
- 为后续技能市场、员工可见范围、审核发布和版本化运行打基础。

## 产品原则

- 用户看到的是“能完成什么任务”，不是 mask、prompt、MCP、插件等工程概念。
- 技能入口要直接进入对应工作区，例如普通对话进入聊天，图片创作进入 SD 工作区。
- 技能配置应服务于任务，配置项包括模型、工具、MCP、可见范围、默认参数、开场问题。
- 不选择技能时仍是普通对话，避免强迫用户先理解技能体系。
- 已开始的会话应固定技能版本，市场更新不应改变历史会话行为。

## 概念边界

| 概念          | 定义                               | 面向对象           |
| ------------- | ---------------------------------- | ------------------ |
| Skill         | 一个用户可选择的任务入口和运行配置 | 用户、管理员       |
| Instructions  | 技能的系统说明、上下文和行为约束   | 模型运行时         |
| Tool          | 技能可调用的具体工具能力           | 模型运行时、开发者 |
| MCP Server    | 工具、资源、提示词的外部能力连接层 | 开发者、管理员     |
| Workspace     | 技能启动后的界面和交互流程         | 用户               |
| Skill Package | 可发布、安装、升级的技能包         | 市场、组织         |

## Skill Package v1

技能包应使用一个 manifest 描述元数据、运行方式、权限和依赖。建议文件名为 `skill.json`，未来可扩展为目录包。

```json
{
  "schemaVersion": "1.0",
  "id": "image.creation",
  "version": "1.0.0",
  "name": {
    "cn": "图片创作",
    "en": "Image Creation"
  },
  "description": {
    "cn": "围绕画面创意、提示词和生成参数来组织一次图片创作。",
    "en": "Create images with prompt, style, model, and generation settings."
  },
  "author": {
    "name": "Yeying",
    "url": "https://example.com"
  },
  "icon": {
    "type": "emoji",
    "value": "1f5bc-fe0f"
  },
  "tags": ["image", "creative"],
  "category": "creation",
  "visibility": {
    "scope": "public",
    "roles": []
  },
  "launch": {
    "type": "workspace",
    "target": "sd"
  },
  "instructions": {
    "type": "inline",
    "content": "你正在执行图片创作任务..."
  },
  "starters": [
    "给我生成一张海报风格的封面。",
    "把这个产品想法转成适合图片模型的提示词。"
  ],
  "model": {
    "syncGlobalConfig": false,
    "default": {
      "provider": "OpenAI",
      "model": "gpt-image-1",
      "endpoint": "/v1/responses"
    },
    "candidates": []
  },
  "tools": [],
  "mcp": {
    "servers": []
  },
  "ui": {
    "entryLabel": "图片创作",
    "emptyState": "minimal"
  },
  "permissions": {
    "network": false,
    "filesystem": false,
    "wallet": false,
    "externalTools": []
  },
  "compatibility": {
    "appVersion": ">=0.1.0",
    "mcp": ">=2025-06-18"
  },
  "release": {
    "status": "published",
    "review": "approved"
  }
}
```

## 字段要求

### 基础信息

- `schemaVersion`：技能规范版本，必填。
- `id`：全局稳定 ID，必填。市场技能建议使用反向域名或命名空间，例如 `community.report.weekly`。
- `version`：语义化版本，必填。
- `name`、`description`：至少包含当前语言，建议支持中英文。
- `author`：发布者信息，市场技能必填。
- `icon`：支持 emoji、内置头像、远程图片三种形式。
- `tags`、`category`：用于发现、筛选和推荐。

### 启动方式

`launch` 决定点击技能后进入哪里。

```ts
type SkillLaunch =
  | { type: "chat" }
  | { type: "workspace"; target: "sd" | "chat" | "settings" | string }
  | { type: "external"; url: string };
```

当前产品建议：

- 普通任务技能使用 `chat`。
- 图片创作使用 `workspace:sd`。
- 后续代码执行、知识库、报表等可以扩展专属 workspace。

### 会话运行模型

技能启动后只有两种产品形态：

- 独立会话页：技能进入专属工作区。适用于图片创作、代码执行、知识库、报表等有明显专属流程的任务。
- 共用会话页：技能进入通用 Chat 页面，但必须带自己的工具条、模型限制、工具依赖和启动条件检查。

```ts
type SkillSessionMode =
  | {
      mode: "workspace";
      target: "sd" | "code" | "knowledge" | "report" | string;
    }
  | {
      mode: "chat";
      toolbar: SkillSessionToolbar;
      requiredModels?: SkillModelConfig["candidates"];
      requiredTools?: SkillTool[];
      requiredMcp?: SkillMcpServer[];
    };
```

运行规则：

- `launch.type = "workspace"` 时，点击技能直接进入对应工作区，不再先创建普通 Chat 会话。
- `launch.type = "chat"` 或未声明 `launch` 时，进入共用 Chat 页面，但工具条必须由技能声明决定。
- 启动前必须检查模型、MCP、工具和权限是否满足；不满足时进入配置页或 MCP 页，并展示明确原因。
- 共用 Chat 只是承载壳，不代表所有技能都使用同一套按钮和参数。
- 技能会话创建后应保存技能快照，避免市场技能更新影响历史会话行为。

### Instructions

`instructions` 是技能运行时给模型的任务说明。它可以是 inline 文本，也可以引用包内文件。

```ts
type SkillInstructions =
  | { type: "inline"; content: string }
  | { type: "file"; path: "SKILL.md" };
```

建议规则：

- 面向任务行为，不写营销文案。
- 明确输入处理、输出格式、需要调用的工具、失败时如何反馈。
- 不要把模型服务商、密钥、环境变量写进 instructions。

### 模型配置

模型配置用于约束该技能的默认模型和候选模型。

```ts
type SkillModelConfig = {
  syncGlobalConfig: boolean;
  default?: {
    provider: string;
    model: string;
    endpoint?: "/v1/responses" | "/v1/messages" | "/v1/chat/completions";
  };
  candidates?: Array<{
    provider: string;
    model: string;
    endpoint?: "/v1/responses" | "/v1/messages" | "/v1/chat/completions";
    reason?: string;
  }>;
  params?: Record<string, unknown>;
};
```

原则：

- 普通对话默认跟随全局模型。
- 专业技能可以设置候选模型。
- 图片创作、语音、嵌入等非聊天技能应明确模型类型和默认参数。

### 端点策略

技能应面向现代模型端点设计，而不是默认绑定旧聊天接口。

```ts
type SkillModelEndpoint =
  | "/v1/responses"
  | "/v1/messages"
  | "/v1/chat/completions";
```

原则：

- OpenAI 兼容模型优先使用 `/v1/responses`。
- Anthropic 模型优先使用 `/v1/messages`。
- `/v1/chat/completions` 只作为旧 provider、旧 Router 或第三方兼容服务的回退路径。
- 如果技能需要多模态输入、工具调用、结构化输出或其他新模型能力，应显式约束候选模型支持对应现代端点。
- 技能市场展示时应把“所需端点能力”作为兼容性信息，而不是让用户在运行失败后再理解。

### 工具和 MCP

技能可以声明工具，也可以声明 MCP Server 依赖。

```ts
type SkillTool = {
  id: string;
  name: string;
  description?: string;
  required: boolean;
};

type SkillMcpServer = {
  id: string;
  name: string;
  transport: "stdio" | "http" | "sse";
  required: boolean;
  configSchema?: Record<string, unknown>;
};
```

原则：

- MCP 是能力连接层，不直接暴露成用户入口。
- 技能详情页必须展示会使用哪些工具和 MCP。
- 工具权限要可审核、可关闭、可按组织策略限制。

### 权限

权限是技能市场和组织分发的信任基础。

```ts
type SkillPermissions = {
  network: boolean;
  filesystem: boolean;
  wallet: boolean;
  externalTools: string[];
  dataScopes?: string[];
};
```

产品展示建议：

- 安装前展示权限摘要。
- 首次运行高风险能力时二次确认。
- 管理员可以限制组织成员安装或运行某些权限的技能。

### 会话界面

技能需要声明自己的会话工具条，而不是默认复用普通对话的全部按钮。

```ts
type SkillSessionToolbar = {
  settings?: boolean;
  theme?: boolean;
  promptHints?: boolean;
  skillSwitcher?: boolean;
  clearContext?: boolean;
  modelSelector?: boolean;
  imageUpload?: boolean;
  imageParams?: boolean;
  plugins?: boolean;
  mcp?: boolean;
  shortcutKeys?: boolean;
  realtime?: boolean;
};
```

产品原则：

- 普通问答可以保留完整工具条。
- 阅读、调研、对比类技能应突出模型、MCP 和提示入口，减少插件、实时语音等无关按钮。
- 深度思考类技能应保持聚焦，默认隐藏图片上传、插件、MCP 和主题切换。
- 图片创作类技能应进入专属工作区，并只暴露模型、图片上传、尺寸、质量等创作相关参数。
- 技能缺少必需模型、MCP 或工具时，不应让用户在会话里试探，应在启动前或工具条区域明确提示。

### 可见范围

```ts
type SkillVisibility = {
  scope: "private" | "organization" | "public";
  roles?: string[];
  users?: string[];
};
```

适用场景：

- `private`：个人自定义技能。
- `organization`：部分员工可见、部门可见、全员可见。
- `public`：市场公开技能。

### 发布状态

```ts
type SkillRelease = {
  status: "draft" | "published" | "deprecated" | "removed";
  review?: "pending" | "approved" | "rejected";
  changelog?: string;
};
```

规则：

- `draft` 只允许作者和管理员可见。
- `published` 可安装。
- `deprecated` 已安装可继续使用，但不推荐新安装。
- `removed` 市场不可见，历史会话仍保留快照。

## 运行时规则

- 会话创建时保存技能快照，包括 `id`、`version`、`instructions`、`model`、`tools`、`launch`。
- 技能更新后不影响历史会话，用户可手动升级会话技能版本。
- 工具不可用时，运行时必须清晰提示，而不是静默降级。
- 技能启动失败时回退到技能详情页或普通对话，并展示失败原因。
- 删除技能不删除历史会话，只阻止继续从市场启动该技能。

## 市场规则

市场技能必须具备：

- 唯一 `id` 和明确版本。
- 中英文至少一种名称和描述。
- 权限声明。
- 启动方式。
- 兼容版本。
- 作者信息。
- 审核状态。

市场列表建议展示：

- 名称、简介、图标、分类、作者。
- 安装量、更新时间、评分。
- 所需权限。
- 支持的工作区和模型类型。

## 与当前实现的兼容关系

当前 `app/store/skill.ts` 中的字段可映射为：

| 当前字段             | v1 字段                                        |
| -------------------- | ---------------------------------------------- |
| `id`                 | `id`                                           |
| `avatar`             | `icon.value`                                   |
| `name`               | `name[currentLang]`                            |
| `description`        | `description[currentLang]`                     |
| `category`           | `category`                                     |
| `starters`           | `starters`                                     |
| `context`            | `instructions`                                 |
| `syncGlobalConfig`   | `model.syncGlobalConfig`                       |
| `modelConfig`        | `model.default` + `model.params`               |
| `candidateModels`    | `model.candidates`                             |
| `plugin`             | `tools`                                        |
| `ui.sessionToolbar`  | `ui.sessionToolbar`                            |
| `launch.type = "sd"` | `launch = { type: "workspace", target: "sd" }` |
| `builtin`            | `source = "builtin"`                           |

兼容策略：

- 保留现有 `Skill` 类型作为运行时兼容层。
- 新增 `SkillPackage` 类型作为发布和市场层。
- 内置技能先从 `SkillPackage` 转换成现有 `Skill`，避免一次性替换所有调用点。
- 用户已有旧技能继续走 migrate，补齐默认 `schemaVersion`、`version`、`visibility`、`permissions`。

## 下一步实施切片

### 第一阶段：规范落地，不改变用户体验

- 新增 `SkillPackage`、`SkillLaunch`、`SkillPermissions`、`SkillVisibility` 类型。
- 增加 `SkillPackage -> Skill` 转换函数。
- 内置技能继续展示原体验，但数据源逐步改为规范结构。
- 给现有技能补默认版本、权限和可见范围。

### 第二阶段：技能详情和配置

- 技能详情页展示模型、工具、MCP、权限、可见范围。
- 管理员可设置组织可见范围。
- 用户可在启动前调整模型和必要参数。
- 图片创作技能明确展示为启动 SD 工作区。

### 第三阶段：市场和版本化

- 支持安装、升级、下架技能。
- 会话保存技能快照和版本。
- 市场列表支持分类、搜索、权限筛选。
- 发布流程加入审核状态和 changelog。

## 当前优先级

建议下一步先做第一阶段：类型和转换层。这样不会打断现有产品流程，但能让后续技能详情、市场、组织权限都基于同一套规范演进。
