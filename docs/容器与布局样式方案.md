# 容器与布局样式方案

本文用于沉淀 Chat 应用当前的页面容器体系、滚动边界和后续布局改造原则，避免新页面继续叠加历史样式约束。

## 1. 背景

当前应用最早是围绕普通 chat 页面设计的。

历史上布局控制分散在以下几层：

- `app/styles/globals.scss`
  - 定义全局尺寸变量，如 `--window-width`、`--window-height`、`--sidebar-width`
- `app/components/home.module.scss`
  - 定义应用主容器、侧边栏、窗口内容区
- 各页面自己的样式文件
  - 例如 `chat.module.scss`、`mask.module.scss`

问题是，这三层过去同时在管尺寸、滚动和边距，导致：

- 新页面很容易受到旧页面容器约束影响
- 工作区页面和普通聊天页会互相争抢滚动权
- 一旦页面结构复杂，就会出现遮挡、溢出、改样式反复的问题

## 2. 当前结构

截至 `2026-05-17`，容器职责开始按层收敛为：

### 2.1 全局变量层

文件：

- `app/styles/globals.scss`

职责：

- 仅保留真正需要全局共享的尺寸变量
- 当前保留：
  - `--window-width`
  - `--window-height`
  - `--sidebar-width`
  - `--message-max-width`
  - `--full-height`

原则：

- 不再在全局层定义页面内容区宽度这类“页面级计算变量”
- 例如 `--window-content-width` 已移除

### 2.2 应用壳层

文件：

- `app/components/home.module.scss`
- `app/components/home.tsx`
- `app/styles/window.scss`

职责：

- 管理应用主壳
- 管理侧边栏与内容区并排结构
- 管理普通卡片壳与全屏壳的切换
- 管理页面通用 header 的收缩边界

当前关键容器：

- `.container`
  - 默认应用卡片壳
- `.container-full`
  - 全屏壳
  - 用于桌面紧凑模式和工作区型页面
- `.window-content`
  - 主内容区，改为 `flex: 1`，不再依赖额外宽度变量
- `.page-content`
  - 统一页面内容壳，接管 `flex / min-height / overflow`
- `.window-header`
  - 通用页面头部
  - 必须允许标题区收缩，不能反向撑开正文区

原则：

- 页面级滚动边界必须先经过 `page-content`
- `window-content` 只负责承接内容，不再负责做宽度计算
- 通用 header 只负责固定头部，不参与正文滚动

### 2.3 页面层

文件示例：

- `app/components/chat.module.scss`
- `app/components/mask.module.scss`

职责：

- 页面自己的内部布局
- 页面自己的滚动区域
- 页面自己的工具条、画布、表单区

原则：

- 页面层只负责自己的内部结构
- 不再回头重新定义应用壳的尺寸语义
- 页面根节点必须显式补齐：
  - `min-width: 0`
  - `min-height: 0`
  - 必要时接管 `overflow`
- 固定工具条、输入条这类区域必须显式 `flex-shrink: 0`
  - 避免反向压缩正文滚动区

## 3. 当前改造原则

后续页面改造遵循以下规则：

1. 新工作区优先走 `container-full`
   - 不要继续套默认 chat 卡片壳
2. 页面滚动权从上到下必须唯一
   - 先 `page-content`
   - 再页面根容器
   - 再页面内部具体滚动区
3. 页面布局不要再依赖额外的“内容区宽度计算变量”
4. 所有会横向溢出的区域必须显式加：
   - `min-width: 0`
   - 必要时加 `overflow: hidden/auto`
5. 不把说明性文字当作布局的一部分
   - 说明文案越多，越容易造成边界问题

## 4. 普通页面骨架

对于 `settings / masks / plugins / search` 这类非工作区页面，统一按以下骨架处理：

- 页面根节点负责：
  - `display: flex`
  - `flex-direction: column`
  - `min-width: 0`
  - `min-height: 0`
- `window-header` 固定在顶部，不参与正文滚动
- 页面 body 负责：
  - `flex: 1`
  - `min-width: 0`
  - `min-height: 0`
  - `overflow: auto`
- 列表卡片里的标题区必须允许收缩
  - 否则右侧操作按钮会把内容区顶出边界
